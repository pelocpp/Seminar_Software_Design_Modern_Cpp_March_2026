Java:
-----

==> Hat nur REFERENZ-SEMANTIK!

    KEIN Problem:     bei "a = b;"  habe ich ZWEI Referenzen,

class Person { ... }

void func()
{
   Person p;                    // p: REFERENZ  // Adresse (Pointer): 8 Bytes

   // Garbage Collector (GC)
}


C++:
-----

==> Hat WERTE-SEMANTIK!     Value-Based

    Problem:     bei "a = b;"  entsteht eine KOPIE


==> Hat REFERENZ-SEMANTIK!  Reference-Based  // 2 Ausprägungen

                                     // mit Adressen / Zeigern:
                                        Konzeptionell: Für Objekte auf dem HEAP (new)

                                     // mit Referenzen
                                        Konzeptionell: Für Objekte auf dem STACK,
                                        um überflüssige KOPIEN zu vermeiden.

                                    // Auch: Bei der Parameterübergabe
                                    // auch hier will ich (meistens) Kopien vermeiden



Worin unterscheiden C++ - und Java Referenzen
sich fundamental:

C++ Referenzen zeigen IMMER auf DASSELBE Objekt !!!








class Person { ... }

void func()
{
    Person p;                // Das ist EIN OBJEKT
}

void func()
{
    //  Reference-Based  // 1 Ausprägung: Mit Zeiger
    //  Diese ist SEHR ÄHNLICH zu Java

    Person* p;               // Das ist EIN Zeiger / Pointer
    p = new Person(...);     // Daten: Heap

    // NO Garbage Collector (GC)
    delete p;
}

    std::vector<std::function<void()>>  m_events;

    // oder

    std::vector<std::move_only_function<void()>>  m_events;

    template<typename TFunc, typename ... TArgs>
    void enqueueTask(TFunc&& func, TArgs&& ...args)
    {

        // using "Generalized Lambda Capture" to preserve move semantics
        auto callable {
            [func = std::forward<TFunc>(func),
            ... capturedArgs = std::forward<TArgs>(args)] () {
                std::invoke(std::move(func), std::move(capturedArgs) ...);
            } 
        };

        {
            // RAII guard
            std::lock_guard<std::mutex> guard{ m_mutex };
            m_events.push_back(std::move(callable));
        }

        m_condition.notify_one();
    }
